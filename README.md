# Axonscut 🎬
**高效、隱私、免上傳的線上影音剪輯工具**

Axonscut 是一個專為快速處理影音需求設計的網站，核心理念是「本地處理」。使用者無需登入、無需將檔案上傳至雲端伺服器，即可直接在瀏覽器進行影音剪輯。

## 🎯 專案目標
* **隱私至上**：檔案不經過伺服器，完全在使用者本地瀏覽器完成運算。
* **即時便利**：打破傳統剪輯軟體需安裝、需登入的門檻，開啟網頁即刻剪輯。

## 🛠 技術挑戰與解決方案 (Troubleshooting)

### 1. COOP/COEP 跨來源隔離限制
* **問題原因**：
    為了防範 Spectre 等資安攻擊，瀏覽器會限制未開啟「跨來源隔離狀態」的環境使用 **共享記憶體** 或 **高精度時間測量** 的功能，這導致 FFmpeg.wasm 等剪輯核心組件無法運作。
* **技術瓶頸**：
    GitHub Pages 為靜態託管，不支援自訂 HTTP Response Headers，無法手動開啟隔離政策。
* **解決方案**：
    將網站部署平台遷移至 **Vercel**，並透過 `vercel.json` 配置自訂 Header：

### 2. 靜態資源體積超出 GitHub 上傳限制
* **問題原因**：
    FFmpeg 相關的二進位檔案（如 `.wasm` 庫）體積較大，若使用 GitHub 網頁版介面（Web UI）直接上傳，會觸發單檔案 **25MB** 的上傳大小限制。
* **解決方案**：
    捨棄網頁端上傳，改由本地端 **Git CLI (Git Bash)** 進行版本控制與推送（`git push`）。
    > **註**：Git 命令列工具支援單檔案最高 100MB 的上傳。

### 3. 影片剪輯/轉檔速度過慢
* **問題原因**：
    在瀏覽器環境下使用 `-c:v libx264` 編碼器會進行「重新編碼」，這對客戶端 CPU 壓力極大，導致單純的剪輯動作卻需消耗過長的運算時間。
* **解決方案**：
    優化 FFmpeg 指令，將「**重編碼**」模式改為「**串流拷貝 (Stream Copy)**」快速模式：
    * **優化前**：`-c:v libx264` (耗時長，重新計算像素)
    * **優化後**：`-c:v copy -c:a copy` (極速，僅重新封裝封包)
    * **結果**：在不改變影片格式與解析度的前提下，剪輯速度提升數倍。
